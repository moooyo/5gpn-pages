安装 · 网关侧

一行命令，装好网关

在一台 root 权限的 Linux 网关上执行。交互式问答（WebUI 域名 / DoT 域名 / 网关 IP），自动签发证书、拉起 systemd 服务。

root@gateway ~ 
$ curl -fsSL https://raw.githubusercontent.com/moooyo/5gpn/main/quick-install.sh | sudo bash

也可在 checkout 内运行 sudo bash install.sh，支持环境变量预置与内网部署。

原理 · 一次解析的旅程

不是隧道，
是一次 DNS 解析决定去向

三类规则自上而下匹配。远端域名被解析成网关自己的 IP，自然漏斗进 mihomo 透明转发；直连域名返回真实 IP，网关不经手。

01

接入

客户端把全部 DNS 查询经 DoT :853 交给网关——唯一入口，证书自动续期、热重载。

02

判定

force-direct / blacklist 优先命中；其余并发查询本地 UDP 与可信 DoT，按 chnroute 仲裁。

03

直连

判定直连的域名返回真实 IP，客户端直连，网关不经手。

04

出站

判定远端的域名解析成网关 IP，流量漏斗进 mihomo，透明转发直出。

chnroute 仲裁按答案 IP 归属判定、而非谁先应答——确定性正是当初自研的原因

去向 · 出口全在应用层

两种去向，多个出口

直连 · DIRECT

真实 IP，本地直出

命中 force-direct 或仲裁判定国内的域名，返回真实 IP——客户端直连，网关完全不经手。

  • ·零额外时延，本地流量零开销
  • ·force-direct 名单随手可加，内网域名精确直连
远端 · 经 MIHOMO

解析成网关 IP，透明转发

远端域名自然漏斗进 mihomo，出站层 selector 切换出口——控制台 / Telegram 一键生效。

  • ✓ 直出
  • 香港
  • 日本
  • 美国
  • 新加坡
  • ·无内核路由与策略路由，切换无需重启

SNI 回源解析器可配置 · 出口列表随 mihomo 出站配置生效

特性

简单，且确定

零 Python、极小依赖，一切行为可预期。

确定性 chnroute 仲裁

并发查询本地 UDP 与可信 DoT，按答案 IP 归属判定直连或直出，不是竞速——这是 smartdns 做不到、当初自研的根本原因。

DoT-only 入口

客户端唯一的 DNS 传输是 DoT :853，Let's Encrypt 证书按文件 mtime 热重载、自动续期；Android / iOS 原生支持。

规则订阅

进程内按各自 interval 定时拉取远程列表，支持 gfwlist / dnsmasq / hosts / cidr 等格式；拉取失败保留旧缓存，离线安全。

统一控制面

Web 控制台、REST API、iOS 描述文件、Telegram bot 共用同一进程内 Controller，浏览器即达。

fail2ban 式防护

登录失败超阈值自动封锁来源 IP；控制台令牌可用 5gpn --rotate-token 一键轮换。

零依赖 · 运维友好

Go 侧仅两个依赖，网关上不放工具链、从不现场编译；systemd 硬化沙箱、统计持久化存活重启、证书续期无需重启服务。

CLI · 一个命令

一个 5gpn 命令
管理一切

安装完成后，终端输入 5gpn 打开交互管理菜单，或带子命令直接执行。配置只有一个文件 /etc/5gpn/dns.env；重跑安装脚本即得全新产物，旧配置错位不可能残留。

gateway ~ 5gpn
$ 5gpn# 打开交互管理菜单
$ 5gpn --status# 服务 / 域名 / 列表状态
$ 5gpn --add-domain d# 强制代理某域名
$ 5gpn change-resolver r# 设置 SNI 回源解析器
$ 5gpn --rotate-token# 轮换控制台令牌
$ 5gpn --ios# 重新生成 iOS 描述文件 + 二维码
