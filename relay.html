Apple Network Relay · MASQUE

无需 VPN，也能分流。

依靠 5GPN 与 Apple Relay 能力，实现按规则的分流访问——装的是描述文件，不是 VPN，状态栏干干净净。

  • 带 UDP/QUIC

原理 · 一个数据包的旅程

它不是 VPN，
是一个按规则工作的系统级中继

iOS 17 起，Apple 把系统级中继背后的 MASQUE 机制开放给了描述文件。整条链路是顺序发生的。

01

解析

先解析目标地址，判断它该本地直连还是交给 5GPN 中继；配套加密 DNS 保证解析稳定可靠。

02

判定

按所选规则匹配目标（自动含子域）：命中直连集就本地直出，否则交给中继。

03

中继

用 CONNECT / CONNECT-UDP（RFC 9298）经专线送到 KFCHost 节点，由节点在出口重新解析并连出——是 MASQUE 而非隧道网卡，状态栏没有 VPN 图标。

04

直连

其余流量根本不进隧道，蜂窝 / Wi-Fi 正常出口，专线只承载真正需要它的流量。

两种模式 · 都支持自定义包含 / 排除

选一条基线规则，再按需微调

黑名单模式

只有命中的走中继

命中名单的目标走 5GPN 中继，其余全部本地直连。

  • ≈150 KB描述文件小，专线只跑该跑的
  • ·省专线、省电，适合日常使用
  • ·名单未覆盖的新目标会走直连（随订阅更新）
白名单模式

未命中的全部走中继

只有命中名单的目标本地直连，其余全部走中继，覆盖最全。

  • ≈350 KB精选约 1 万个主流直连域名，可在 iOS 安装
  • ·目标在 KFCHost 节点解析，链路最干净
  • ·新目标默认走中继不会漏，适合全量分流

规则名单随 5GPN 订阅包每日更新分发，可在生成器里叠加自定义包含 / 排除

开始使用 · 三步

配合 KFCHost 与 5GPN 订阅包

5GPN Relay 负责分流与描述文件；专线出口与规则订阅由 KFCHost 与 5GPN 订阅包提供。

1

开通 KFCHost 专线主机

在 KFCHost 开通一台经专线可达的主机，作为 5GPN 的专线出口节点。

前往 KFCHost 官网 ›
2

订阅 5GPN

获取 5GPN 订阅包与访问令牌。规则名单每日更新，随订阅自动分发。

3

生成并安装配置

在生成器填入域名与令牌，选模式、加规则，一键安装到 iPhone。

打开生成器 ›

安装 · KFCHost 专线主机

一条命令拉起中继服务器

在你的 KFCHost 专线主机上（Debian / Ubuntu / CentOS / Rocky，root）运行。它会装好 Docker、gum、Node，签证书、起 Envoy，并生成 iOS 配置 + 二维码。

root@kfchost-box 
$ curl -fsSL https://raw.githubusercontent.com/moooyo/5gpn-relay/main/install.sh | sudo bash

需要一个指向 KFCHost 主机、经专线可达的域名。装完后运行 relayctl profiles 生成配置并出二维码，或用网页生成器自定义规则。

5GPN Relay

基于 Apple Network Relay（MASQUE）把 iOS 流量按规则做内网分流，经专线直达 KFCHost 节点——无 VPN 图标。需配合 KFCHost + 5GPN 订阅包。

未签名描述文件安装时显示「未验证」，自用可接受。

令牌即权限：泄露 = 开放代理，请用 relayctl rotate 轮换。
